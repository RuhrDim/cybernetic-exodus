# Пошаговая инструкция: как опубликовать книгу на GitHub

*Написано для тех, кто никогда не работал с GitHub. Читай медленно, делай по шагам.*

---

## Что такое GitHub и зачем он нужен

GitHub — это сайт где хранятся текстовые файлы (и программный код), и где люди могут предлагать правки. Представь это как Google Docs, но для проектов с открытым доступом. Каждый желающий может предложить исправление, а ты решаешь принять его или нет.

---

## Шаг 1 — Зарегистрироваться

1. Открой сайт **github.com**
2. Нажми кнопку **Sign up** (Регистрация) в правом верхнем углу
3. Введи email, придумай имя пользователя (например `cybernetic-exodus`) и пароль
4. Подтверди email (придёт письмо)

---

## Шаг 2 — Создать репозиторий (папку проекта)

1. После входа нажми зелёную кнопку **New** (или **+** → **New repository**)
2. В поле **Repository name** напиши: `cybernetic-exodus`
3. В поле **Description** напиши: `The Architecture of Post-Capitalist Civilization`
4. Выбери **Public** (чтобы все могли читать)
5. Поставь галочку **Add a README file**
6. Нажми зелёную кнопку **Create repository**

Репозиторий создан. Теперь у тебя есть адрес вида:
`https://github.com/твоё-имя/cybernetic-exodus`

---

## Шаг 3 — Загрузить файлы

У тебя есть архив `cybernetic_exodus_github.zip`. Вот как загрузить его содержимое.

### Способ А — Через браузер (самый простой)

1. Открой свой репозиторий на GitHub
2. Нажми кнопку **Add file** → **Upload files**
3. Перетащи файлы из архива прямо в браузер
4. **Важно**: загружай папки по одной. Сначала `en/book1/`, потом `en/book2/` и так далее
5. Внизу страницы нажми зелёную кнопку **Commit changes**

> ⚠️ GitHub через браузер не любит целые папки сразу. Если не получается — используй Способ Б.

### Способ Б — Через GitHub Desktop (удобнее)

1. Скачай программу **GitHub Desktop** с сайта `desktop.github.com`
2. Войди в свой аккаунт GitHub
3. Нажми **Clone repository** → выбери `cybernetic-exodus`
4. Программа создаст папку на твоём компьютере
5. Скопируй все файлы из архива `cybernetic_exodus_github.zip` в эту папку
6. В программе GitHub Desktop нажми **Commit to main** → **Push origin**

Файлы загружены.

---

## Шаг 4 — Заменить README.md

После загрузки всех файлов:

1. Открой файл `README.md` в репозитории (нажми на него)
2. Нажми иконку карандаша (Edit)
3. Удали всё что там было при создании
4. Вставь содержимое файла `README.md` из архива
5. Нажми **Commit changes**

---

## Шаг 5 — Включить красивый просмотр (необязательно, но красиво)

GitHub умеет показывать Markdown-файлы как красиво отформатированный текст. Это работает автоматически — просто открой любой `.md` файл и увидишь.

Если хочешь сделать сайт из книги:
1. Зайди в **Settings** репозитория
2. Найди раздел **Pages**
3. В разделе **Source** выбери **Deploy from a branch**
4. Выбери ветку **main** и папку **/ (root)**
5. Нажми **Save**

Через несколько минут появится адрес вида `твоё-имя.github.io/cybernetic-exodus` — это уже настоящий сайт.

---

## Шаг 6 — Как люди будут предлагать правки

Когда кто-то хочет исправить главу:
1. Он открывает нужный файл (например `en/book1/04_ch1_marxist_audit.md`)
2. Нажимает карандаш (Edit)
3. Вносит правку
4. Нажимает **Propose changes**
5. Ты получаешь уведомление на email
6. Заходишь в раздел **Pull requests** и видишь что именно изменилось
7. Нажимаешь **Merge** (принять) или **Close** (отклонить)

Всё. Ты всегда контролируешь что попадает в книгу.

---

## Как поделиться ссылкой

После публикации просто отправляй людям ссылку:
`https://github.com/твоё-имя/cybernetic-exodus`

Или если сделал сайт через Pages:
`https://твоё-имя.github.io/cybernetic-exodus`

Открой раздел **Issues** в своём репозитории и напиши мне туда — или возвращайся сюда, разберёмся.
